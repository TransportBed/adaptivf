# Paper Exports

Canonical generated export bundle for the finished AdaptIVF paper run.

This directory is designed to be publishable: it contains the normalized study summaries, paper-ready plots, table CSVs, and serving-RAM measurements. The directory is regenerated by the main pipeline rather than maintained manually.

## What Is Here

- `initialization/`, `competitiveness/`, `ablations/`: normalized per-study CSV/JSON summaries
- `tables/`: compact CSV tables used by the paper
- top-level `*.pdf` / `*.png` / `*.csv`: paper and appendix plots with their plot-source CSVs
- `serving_ram.json` and `serving_ram.csv`: isolated subprocess serving-RSS measurements for the canonical competitiveness experiments
- `run_plan.json`, `study_manifest.json`, `plots_manifest.json`, `tables_manifest.json`: machine-readable manifests for the generated bundle
All manifests and artifact-provenance paths in this bundle are stored as
repo-relative paths rather than machine-local absolute paths.

Study row counts in this finished bundle:

- initialization: 48 rows
- competitiveness: 44 rows
- ablations: 24 rows

## Reproduce This Directory

From the repository root:

```bash
bash scripts/download_datasets.sh
bash scripts/run_paper.sh
```

If the experiment artifacts already exist and only the export directory needs to be rebuilt:

```bash
PYTHONPATH=src python -m collect --out-root paper_exports --experiments-root ../experiments/adaptivf --data-root ../data
PYTHONPATH=src python -m tables --out-root paper_exports
PYTHONPATH=src python -m plots --out-root paper_exports
PYTHONPATH=src python -m export_bundle --out-root paper_exports
```

The commands above are expected to recreate the same canonical export layout and filenames shown here. They do not promise byte-identical PDFs or identical runtime-sensitive values such as QPS, query-RAM deltas, or serving RSS, which can vary across machines and reruns. The root repository README gives the high-level workflow; see `scripts/README.md` for the runnable command surface. The maintained supplementary-material source now lives in `docs/appendix.tex`.

## Notes

- All results are CPU-only and were generated without GPU acceleration.
- `index_overhead_mb` is the primary compactness metric.
- `avg_computations` is the canonical workload metric.
- `query_mem_delta_mb` reports isolated subprocess query-path RSS deltas.
- `rss_serving_mb` reports isolated subprocess serving RSS after loading the artifacts required for query serving.
- The exported RAM figures intentionally separate query-path RSS deltas from post-load serving RSS.
