\section{Experimental Setup}
\label{sec:appendix:setup}

All experiments were conducted on a single server with the following
specification:

\begin{itemize}
  \item \textbf{CPU}: AMD EPYC 9124 16-Core Processor (64 logical threads)
  \item \textbf{RAM}: 1.5\,TiB DDR5
  \item \textbf{Disk}: 17\,TB NVMe
  \item \textbf{GPU}: None (all computation is CPU-only)
  \item \textbf{OS}: Linux (kernel \placeholder{X.Y.Z})
  \item \textbf{Python}: 3.13.9, TensorFlow 2.20.0, FAISS (CPU), PyTorch 2.9.1 (CPU fallback)
\end{itemize}

Thread parallelism is maximised for every library:
OMP\_NUM\_THREADS\,=\,MKL\_NUM\_THREADS\,=\,OPENBLAS\_NUM\_THREADS\,=\,TF\_NUM\_INTRAOP\_THREADS\,=\,64.
TF\_NUM\_INTEROP\_THREADS is set to~2 (standard practice to avoid
over-subscription when intra-op already saturates all cores).
All experiments use seed\,=\,0 for reproducibility.

\paragraph{LIRA configuration.}
LIRA~\cite{lira} uses the published architecture with $B$ buckets (matching
the dataset partition count), a threshold sweep from 0.1 to 1.0 in steps
of 0.02 (46 repartition stages), probe depth 10, and flat inner indexes.
Seed is set to~0 for all datasets.

\paragraph{Entropy-adaptive probing formula.}
For each query $q$, the router computes softmax entropy $H(q)$ over the
partition logits.  The per-query probe count is:
\begin{equation}
m(q) = \left\lfloor m_{\text{base}} + \lambda \cdot \bigl(e^{H(q)/\ln B} - 1\bigr) \right\rfloor
\label{eq:entropy_probing}
\end{equation}
clamped to $[m_{\text{base}},\, m_{\max}]$.  Easy queries (low entropy, confident
router) probe as few as $m_{\text{base}}=5$ partitions; hard queries (high
entropy) probe up to $m_{\max}=80$.

\section{Research Questions and Hypotheses}
\label{sec:appendix:rqs}

competitiveness experiments.  All claims are evaluated on the four datasets

% ---- RQ1 ----
\subsection{RQ1: Does entropy-adaptive probing recover recall lost by static routing?}
\label{sec:appendix:rq1}

\paragraph{Problem statement.}
Given the same IVF partition layout ($B\!=\!1024$ for Glove/Sift/Gist,
$B\!=\!4096$ for Deep1M), confidence-threshold multi-assignment
($\tau\!=\!0.75$, $A\!=\!3$), and single repetition, does entropy-adaptive
probing ($m_{\text{base}}\!=\!5$, $m_{\max}\!=\!80$, $\lambda\!=\!10$) yield
higher Recall@10 than static fixed-depth probing across all four datasets?

\paragraph{Null hypothesis $\boldsymbol{H_0}$(RQ1).}
The mean Recall@10 of AdaptIVF (entropy-adaptive) is less than or equal to
that of AdaptIVF-Static (fixed probe depth):
\[
  H_0: \mu_{\text{Recall}}(\text{AdaptIVF}) \;\leq\; \mu_{\text{Recall}}(\text{AdaptIVF-Static})
\]

\paragraph{Acceptance criterion (reject $H_0$).}
AdaptIVF recall exceeds AdaptIVF-Static recall on \emph{all four} datasets,
with minimum per-dataset improvement $\geq 0.5$ percentage points (pp).

\paragraph{Rejection criterion (fail to reject $H_0$).}
AdaptIVF recall fails to exceed AdaptIVF-Static on any one dataset, or no
single dataset shows $\geq 0.5$\,pp improvement.

\subsection{RQ2: Does adaptive probing impose a disproportionate throughput cost?}
\label{sec:appendix:rq2}

\paragraph{Problem statement.}
Adaptive probing scans more candidates for hard queries, increasing mean
computation per query.  We quantify the throughput price (QPS decrease) paid
per unit of recall gained when switching from AdaptIVF-Static to AdaptIVF,
under single-threaded query evaluation on 64-core AMD EPYC 9124 (1.5\,TiB
RAM, no GPU).

\paragraph{Null hypothesis $\boldsymbol{H_0}$(RQ2).}
The relative QPS reduction exceeds 50\% on at least one dataset:
\[
  H_0: \exists\, d : \frac{\text{QPS}_{\text{Static}}(d) - \text{QPS}_{\text{AdaptIVF}}(d)}{\text{QPS}_{\text{Static}}(d)} > 0.50
\]

\paragraph{Acceptance criterion (reject $H_0$).}
The relative QPS reduction is ${<}\,50\%$ on every dataset.

\paragraph{Rejection criterion (fail to reject $H_0$).}
Any dataset exhibits ${\geq}\,50\%$ QPS reduction.

% ---- RQ3 ----
\subsection{RQ3: Does raising the multi-assignment limit from $A\!=\!3$ to $A\!=\!4$ yield diminishing returns?}
\label{sec:appendix:rq3}

\paragraph{Problem statement.}
AdaptIVF assigns each training vector to up to $A$ partitions when the
router's softmax confidence is below $\tau$.  Increasing $A$ enriches the
candidate pool but inflates inverted lists and per-probe computation.  The
ablation compares AdaptIVF ($A\!=\!3$) vs.\ AdaptIVF-A4 ($A\!=\!4$) with all
other hyper-parameters held constant.

\paragraph{Null hypothesis $\boldsymbol{H_0}$(RQ3).}
AdaptIVF-A4 achieves strictly higher Recall@10 than AdaptIVF by at least
0.5\,pp on at least one dataset:
\[
  H_0: \exists\, d : \text{Recall}(\text{A4}, d) - \text{Recall}(\text{A3}, d) \geq 0.005
\]

\paragraph{Acceptance criterion (reject $H_0$).}
On no dataset does AdaptIVF-A4 improve recall by $\geq 0.5$\,pp over
AdaptIVF, confirming diminishing returns; $A\!=\!3$ is sufficient.

\paragraph{Rejection criterion (fail to reject $H_0$).}
At least one dataset shows $\geq 0.5$\,pp gain from $A\!=\!4$.

% ---- RQ4 ----
\subsection{RQ4: Does adaptive probing mitigate PQ recall degradation?}
\label{sec:appendix:rq4}

\paragraph{Problem statement.}
Product Quantisation (PQ, $m_{\text{PQ}}\!=\!16$, 8 bits) compresses
inverted-list vectors, reducing index footprint at the cost of approximate
distance scoring.  If adaptive probing visits more lists for hard queries,
re-ranking more candidates might compensate for PQ approximation error.  The
ablation provides three matched PQ/non-PQ pairs: Static/Static+PQ,
AdaptIVF/AdaptIVF+PQ, and AdaptIVF-A4/AdaptIVF-A4+PQ.

\paragraph{Null hypothesis $\boldsymbol{H_0}$(RQ4).}
The recall penalty from PQ is the same or larger under adaptive probing than
under static probing on all datasets:
\[
  H_0: \forall\, d : \Delta_{\text{PQ}}(\text{Adaptive}, d) \geq \Delta_{\text{PQ}}(\text{Static}, d)
\]
where $\Delta_{\text{PQ}}(s, d) = \text{Recall}(s, d) - \text{Recall}(s{+}\text{PQ}, d)$.

\paragraph{Acceptance criterion (reject $H_0$).}
The PQ recall penalty under adaptive probing is strictly smaller than under
static probing on at least 3 of 4 datasets.

\paragraph{Rejection criterion (fail to reject $H_0$).}
The adaptive PQ penalty equals or exceeds the static penalty on 2 or more
datasets.

\subsection{RQ5: Does AdaptIVF achieve Pareto-dominant recall--overhead across all scales?}
\label{sec:appendix:rq5}

\paragraph{Problem statement.}
The competitiveness study evaluates nine methods spanning three families
(graph: HNSW; inverted-file: IVF, IVFPQ; learned: BLISS, MLP-IVF, LIRA,
AdaptIVF, MLP-IVFPQ, AdaptIVF+PQ) on datasets from 1M to 10M vectors, with
dimensionalities 96--960 and both L2 and cosine metrics.  A method is
\emph{Pareto-dominant} on the (Recall@10, index overhead) plane if no other
method simultaneously achieves $\geq$ recall and $\leq$ overhead.

\paragraph{Null hypothesis $\boldsymbol{H_0}$(RQ5).}
There exists a baseline and dataset where the baseline dominates AdaptIVF:
\[
  H_0: \exists\, B,d : \text{Recall}(B,d) \geq \text{Recall}(\text{AdaptIVF},d) \;\wedge\; \text{Overhead}(B,d) \leq \text{Overhead}(\text{AdaptIVF},d)
\]

\paragraph{Acceptance criterion (reject $H_0$).}
On every dataset, no single baseline achieves both $\geq$ recall and $\leq$
overhead.

\paragraph{Rejection criterion (fail to reject $H_0$).}
At least one baseline on at least one dataset dominates AdaptIVF on both
axes.

\paragraph{Assumptions and limitations.}
\begin{itemize}
  \item All results are from a single seed ($s\!=\!0$) and a single server.
        Variance across seeds is not measured; conclusions are conditioned on
        this specific training trajectory.
  \item QPS is measured end-to-end (including query preprocessing) but excludes
        index loading time.  Query batches are evaluated sequentially (no
        intra-query parallelism for learned methods; FAISS search uses
        OpenMP).
  \item ``Index overhead'' excludes the shared base-vector store, which all
        IVF-family methods require.  Total deployment cost should add the
        raw data payload ($N \times d \times 4$ bytes for float32).
  \item LIRA trains with TensorFlow in v1-compat mode (CPU only, no GPU).
        Training timings would differ substantially on GPU hardware.
  \item The entropy-adaptive probing formula
        (Eq.~\ref{eq:entropy_probing}) uses a fixed $\lambda\!=\!10$
        across all datasets.  Per-dataset tuning of $\lambda$ could shift
        the recall--throughput trade-off but is not explored in this work.
\end{itemize}
